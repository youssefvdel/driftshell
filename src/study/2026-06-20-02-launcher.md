# 2026-06-20 18:00 — Launcher Overlay Implementation

## Context
Need an app launcher overlay that appears on keybinding. Must coexist with the existing bar.

## Architecture Decision: daemon mode

The bar uses `application()` (single window). For a launcher + bar simultaneously, need `daemon()` mode:
- `daemon()` view: `fn view(&self, window: window::Id) -> Element<Message>` — switch on ID
- `#[to_layer_message(multi)]` generates multi-window messages: `NewLayerShell`, `RemoveWindow`, etc.

### Window IDs
- `window::Id::from(0)` — bar (top anchor, 36px)
- `window::Id::from(1)` — launcher overlay (centered, sizes to content)

### Launcher interactions
- Toggle: `Message::ToggleLauncher` → spawns/removes window ID 1
- Search: text input at top, results below
- Result click: `spawn(app_id)` via driftwm IPC `Action`

## Desktop File Discovery
- Paths: `$XDG_DATA_HOME/applications/` + `$XDG_DATA_DIRS/applications/`
- Parse `.desktop` files for: `Name`, `Exec`, `Icon`, `NoDisplay`, `Type`
- Filter: only `Type=Application`, `NoDisplay=false`
- Cache: load on start, store in `HashMap<String, AppEntry>`
- Use freedesktop `xdg` crate for XDG dirs, manual parser for `.desktop`
